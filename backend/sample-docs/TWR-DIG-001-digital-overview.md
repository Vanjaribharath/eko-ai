## 1. What Digital Does

Digital covers cloud-native application development, full-stack engineering, API and microservices development, DevOps/platform engineering, and modern front-end engineering for client digital products and platforms.

## 2. Common Technologies Used

1. Backend: Java/Spring Boot, Node.js, Python, .NET, depending on client stack.
2. Frontend: React, Angular, Vue.
3. Cloud: AWS, Azure, GCP — Digital is the tower most likely to hold cloud certifications (e.g., AWS Certified, Azure Certified) as a job requirement.
4. DevOps: Docker, Kubernetes, Jenkins, GitHub Actions, Terraform.

## 3. Mandatory Tower Trainings

See Mandatory Trainings document (TRN-MAND-009), Digital section: Secure Coding Practices / OWASP Top 10 overview (required within 14 days) and Cloud Security Fundamentals (required within 30 days).

## 4. Ways of Working

1. Digital projects typically follow Agile/Scrum with 2-week sprints, daily stand-ups, sprint planning, retrospectives, and CI/CD-driven delivery.
2. Code review and pull request approval is mandatory before any merge to a shared branch — enforced via branch protection rules in most projects.
3. Architecture decisions for significant changes go through a lightweight Architecture Decision Record (ADR) process, reviewed by the project's Solution Architect.

## 5. Who to Contact

1. Tower-specific technical questions: your Tech Lead or Solution Architect.
2. Tower allocation/staffing questions: Digital Resource Manager.
3. Cloud certification reimbursement: check the current Learning Reimbursement Policy with your HRBP; Digital tower often has dedicated certification budgets per project.
