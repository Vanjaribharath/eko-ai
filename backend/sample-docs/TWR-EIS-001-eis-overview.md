## 1. What EIS Does

EIS (Enterprise Information Systems) handles large-scale enterprise resource planning (ERP) implementations, data integration, middleware, and business systems that run core client operations — finance, supply chain, HR systems, and cross-system data flows.

## 2. Common Technologies Used

1. ERP platforms: SAP (S/4HANA, ECC), Oracle ERP Cloud, Microsoft Dynamics 365, depending on client.
2. Integration/middleware: MuleSoft, SAP PI/PO, Boomi, Azure Integration Services.
3. Data platforms: SAP BW, Oracle Data Integrator, Snowflake (client-dependent).

## 3. Mandatory Tower Trainings

See Mandatory Trainings document (TRN-MAND-009), EIS section: ERP Systems Overview (required within 14 days) and Data Integration and Middleware Security (required within 30 days).

## 4. Ways of Working

1. EIS projects often follow a hybrid Waterfall-Agile model due to ERP implementation methodologies (e.g., SAP Activate) requiring structured phase gates alongside iterative configuration sprints.
2. Most EIS projects involve a Functional Consultant + Technical Consultant pairing per module (Finance, Supply Chain, HR, etc).
3. Data migration activities require extra care around client data confidentiality, particularly during testing phases that use copies of production data — see IT Security Policy (POL-SEC-003), Section 5 on Data Classification.

## 5. Who to Contact

1. Tower-specific technical questions: your Module Lead or Solution Architect.
2. Tower allocation/staffing questions: EIS Resource Manager.
3. Tower-specific certification reimbursement (e.g., SAP certifications): check with your HRBP for the current Learning Reimbursement Policy.
