About

What is EKO-AI?

Your internal knowledge assistant — searches approved policy documents, gives sourced answers, works fully offline, never guesses.

In one sentence

You ask a question in plain English; EKO-AI finds the most relevant sections across all approved policy documents and returns a direct, structured answer with the exact source attached — with zero external API calls, zero data leaving this machine.

Security Design

Disconnect WiFi after startup and EKO-AI still works perfectly. There is no outbound network call anywhere in the answer path — verifiable by anyone who reads the source code.

Why it exists

Existing tools like Rovo AI and Confluence search return one generic result at a time and only serve the DevOps team. EKO-AI serves all five towers — AuthEnq, EIS, ITS, Digital, and Operations — with accurate, policy-sourced answers combining information from multiple documents at once.

Key capabilities

  • Understands plain English — including typos and abbreviations like "PF", "WFH", "BGV", "F&F"
  • Combines information from multiple policy documents into one structured answer
  • Cites the exact document ID and section for every point in the answer — click any source to read the full policy
  • Responds to greetings, clarification questions, and "what can you do" queries naturally
  • Explicitly refuses to guess when no approved source covers the question
  • Logs every question to an audit trail — timestamp, source docs used, confidence
  • Sub-2ms search latency — answers appear almost instantly

What it does not do

  • Does not answer questions from general internet knowledge — only from approved documents
  • Does not call any external API or AI service — everything runs locally
  • Does not store conversation history after the server restarts (prototype limitation)

How to Use

Running and using EKO-AI locally

Complete steps to start the system, add documents, and get the best answers.

Start the system

  • 1
    Open a terminal in the backend folder and install dependencies (one time only).
  • 2
    Start the server. It automatically reads and indexes all documents in sample-docs/ on startup.
  • 3
    Open this file (frontend/index.html) in any browser — no build step needed.
  • 4
    Ask questions in plain English. Answers arrive in under 100ms.
# Step 1 — install (run once) cd backend pip install -r requirements.txt # Step 2 — start server uvicorn main:app --reload --port 8000 # Step 3 — open frontend (just double-click or drag into browser) frontend/index.html

Adding a new document

# Drop any PDF, Word (.docx), or Markdown (.md) into sample-docs/ # Then reindex without restarting: curl -X POST http://localhost:8000/reindex

Tips for better answers

  • Ask in full sentences — "how many days of earned leave do I get" works better than just "leave days"
  • Common abbreviations work — PF, WFH, BGV, F&F, HRA, LOP, SSO, MFA are all understood
  • Typos are corrected automatically — "leeve", "photu", "clent" are all handled
  • Click any source chip under an answer to read the full policy document
  • If an answer feels incomplete, try adding more context — "I am a new joiner in EIS tower, what trainings do I need?"

Test endpoints (browser)

  • localhost:8000/docs — interactive API explorer (try any endpoint without code)
  • localhost:8000/audit — see every question asked, logged in real time
  • localhost:8000/documents — full document index with chunk counts
  • localhost:8000/health — server status and index statistics
